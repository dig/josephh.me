Discovery Log

Week 1 - Introduction

  • We got given a virtual machine hosted by the university which can be accessed at https://uop-1-server-per-student-prod.appspot.com/instance/get, I can now use a SSH client such as putty or termius (for mac) to connect into the VM and start controlling it.
  • We also got access to the draft coursework so we can start preparing for that.
  • I already use and had visual studio code installed before coming to the university so I kept on using the editor that I was used to however we were told alternatives such as Atom, Notepad++ & Sublime Text which at some point I should check out.

Week 2 – Web Security

  • This week we looked into information & social engineering, we started by listing all our personal information we generate on a daily basis. We then went on to find out how our information was being processed.

Week 3 – Reviewing Literature

  • Since our coursework was due soon, this week we learnt about reviewing literature at university level. I learnt that a literature review should be unbiased, objective and reliable information. The type of papers we should read were academic journals, books and online documents.
  • When we are selecting papers for our writeup, we should skim read papers those topics are similar to the topic we are writing about. Once we are happy with the content of each paper, fully read it and take down structured notes.
  • The main layout of the writeup was introduction, themes, conclusion then reference list.

Week 4 – Academic Writing

  • This week was about improving our writing to suit the university writing standards.
  • Our writing needed to be evidence based, evaluative, thoughtful, logical, neutral and structured which was explained in depth during the lecture.
  • We needed to avoid personal opinions, overuse of brackets and exclaiming.

Week 5 – Introduction to Markup

  • This week we were introduced to the markup language HTML. This is the template HTML we used to start our first HTML file.
  • After creating our index.html file, we uploaded it to our VM by using an FTP client such as FileZilla using the SFTP protocol. We then started a http server on the VM by typing http-server which creates a development server on port 8080 of the VM’s external IP address.
  • After playing around with the basic HTML elements, I went to https://www.w3schools.com and went through all the different HTML elements to refresh my memory since I have previous experience with web development prior to university.

Week 7 – Separating Form and Content

  • The lecture and lab this week were focused towards style. This is an important topic because style is the main part of frontend website development.
  • We learnt about cascading style sheets (CSS) and that we can use the HTML link tag to include our external CSS files into the HTML document.

Week 8 – Productivity and Efficiency: Editors

  • This week we were introduced to different techniques that can make us more efficient while we create our websites.
  • We started to look at all the different packages made for our editors.
    • Live HTML Preview: Allows us to see our changes to the HTML file on the fly, it automatically refreshes the HTML file as we make changes and allows a live preview.
    • Multi Cursor: Allows us to select multiple parts of the document at the same time, this can save us time by allowing us to mass replace fast.
    • Linter: This analyses our code and flags any errors to us.
  • Another method of being more productive was to setup a WinSCP folder sync so whenever we make changes locally it would upload the edited file to our VM automatically and deploy on our development http server. This allows for rapid testing and prevents us from dragging and dropping into a FTP client every time we make a change.

Week 9 – Design

  • This week we were introduced to design techniques that websites use.
  • Every website you create, should start off with a design on a program such as photoshop or paint.
  • A common design pattern is called the MVC pattern means model, view, controller.
    • Model: Data structure, could be a database or file.
    • View: The display part of the pattern, this is usually the HTML and other client sided applications such as JavaScript.
    • Controller: Updates the models and views, it’s the backend of the website.

Week 10 – Using Images

  • This week we were introduced to using the image html tag, this allows us to display images in our html document.
  • I learnt that we can also use the picture tag to display different image types depending if the browser supports it or not.
  • We can also attach a link to the image by using the anchor tag wrapped around the image tag.

Week 13 – Addressing the Web

  • This week was all around linking and URLs on the web, we found that html documents can be navigated through links/anchor tag on pages.
  • Example: http://192.168.0.1:8080/index.php
    • Scheme: The scheme indicates what format in which the request should be sent, some examples are https, ftp, http, mailto.
    • Domain/Host: This part can either be a domain name (e.g google.com) or an IP such as IPv4 or IPv6. Every computer on the internet has a unique identifier known as the IP. Domains are translated to IP through a DNS resolver (e.g 1.1.1.1 or google’s 8.8.8.8).
    • Port: This part is known as the port, computers run services on network ports and allows you choose which service on that host you would like to access. HTTP runs on port 80, HTTPS runs on 443.
    • File: This part tells the server what you would like to access, index.php is used as the default file for PHP webservers.

Week 14 – Website Marking Scheme

  • This week was all about validating our websites using the W3 validator, you attach the W3 validator link as an anchor tag on your site and you can click it to see any errors or warnings about your HTML mark-up.

Week 15 – Standards

  • This week was about internet standards, ISO for international organization for standardization which is the org that deals with setting and updating standards.
  • HTTP, HTML, TCP/IP are all standards but just for different purposes.

Week 16 – Usability

  • This week we learnt how important usability is for all people, every user should gain the same experience and information from your website.
  • General guidelines
    • Handle errors gracefully.
    • If the system is busy, tell the user.
    • Use consistent terms.
    • Follow industry standards.
    • Always consider needs of your user.
  • Usability mistakes
    • Neglecting user aims.
    • Neglecting mobile users.
    • Inconsistent design.
    • Ignoring design standards/conventions.
    • Old content.
    • No search functions.
